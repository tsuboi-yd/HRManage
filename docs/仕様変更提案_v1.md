# 仕様変更提案 v1（Excel様式ベース全面改修）

## 1. 変更概要

現行の人事ヒアリング用Excelシート（様式2〜6 + 元名簿 + 戦力外）の機能を
本システムに統合し、部長・センター長が直接入力する運用に変更する。

---

## 2. 組織階層の拡張

### 現行
```
本部 → センター → 部（3階層）
```

### 変更後
```
本部 → センター → 部 → 課（4階層、課は任意）
```

- 課の追加・編集は管理者（人事）が行う
- 部長は定員入力時に課単位で入力するか部単位で入力するか選択可能
- 課単位入力の場合、部の定員は課の合計から自動集計

### 影響
- Organizationモデルに `level: 'section'` を追加
- 定員・異動計画の入力UIに課選択ドロップダウンを追加

---

## 3. 定員カテゴリの変更（様式2対応）

### 現行
管理職 / 一般正社員 / 非正規 の3区分

### 変更後（Excel様式2準拠）

| 行 | 意味 | 入力/自動 |
|----|------|----------|
| **定員** | あるべき人数 | **部長入力** |
| **現人員（正社員）** | 実際の正社員数 | 自動（元名簿から集計） |
| **人員（正社員以上）** | 正社員＋管理職 | 自動集計 |
| **純粋過不足** | 定員 − 現人員 | 自動計算 |
| **管理職（経営職）** | 管理職の人数 | 自動（群類=管理職で集計） |
| **充員予定（新卒+キャリア）** | 採用計画の合計 | 自動（採用計画から集計） |
| **バイネーム確定済** | 異動計画で名前が入っている人数 | 自動集計 |
| **バイネーム未確定** | 定員−現人員−バイネーム確定数 | 自動計算 |

### 対象期間
- 現在（基準日） / 当年度末 / 翌年度末 / 翌々年度末
- 現行の3年分を維持しつつ、各年度内で4月時点と3月末時点を区別

### バイネーム未確定の表示
- 定員サマリのテーブルではなく、異動計画一覧画面のヘッダーに
  「2026年度: 2名 異動者未確定」のようなアラート表示を追加

---

## 4. 定員変動（様式3対応）

### 新機能
定員数の変更がある場合、その理由を記録する仕組みを追加。

| 項目 | 入力者 |
|------|--------|
| 変動時期 | 部長 |
| 対象部課 | 部長 |
| 変動件名（理由） | 部長（自由記述） |
| 変動人数（±） | 部長 |
| スペック | 部長 |
| 摘要 | 部長（自由記述） |

- 定員を変更した時に「なぜ変更したか」を記録するモーダルを表示
- 履歴として保存し、センター長・人事が確認可能

### 現行の「採用計画」を拡張

| 項目 | 現行 | 追加 |
|------|------|------|
| ポジション | ○ | — |
| 配属先 | ○（自部署固定） | 課まで選択可能に |

## 6. 新卒採用の詳細（様式5-1, 5-2対応）

### 2段階入力方式

**Step1: 基本情報（部長が最初に入力）**
- 人数 / 配属先 / 採用時期 / 技術職or事務職 / 充員理由 / 採用区分

**Step2: 詳細スペック（部長が後から補完）**
- 事業部門 / 領域
- 専攻 / 必要スペック補足
- 学歴（B/M/D、NGの場合理由明記）
- 語学力 / 資格
- 勤務地
- 職場環境について
- 特記事項

### マスタ管理（プルダウン用）
以下をマスタテーブルで管理し、管理者が編集可能にする：

| マスタ | 例 |
|--------|-----|
| 事業部門 | マテリアル / エレクトロニクス / 建材 / ファーマ / ライフサイエンス 等 |
| 領域 | エッセンシャルケミカル / コンフォートライフ / セパレータ 等 |
| 勤務地 | 東京 / 大阪 / 神ヶ浦 / 名古屋 / 札幌 / 福岡 等 |
| 専攻 | 電子 / 機械 / 化学 / 生物 / 材料 等 |
| 採用区分 | 既存事業維持 / 事業拡大 |
| 学歴 | 学士(B) / 修士(M) / 博士(D) / 不問 |

## 8. 戦力外管理（様式6対応）

### 理由区分マスタ

| 区分 | 定義 |
|------|------|
| ①健康・メンタル不調によるパフォーマンス不良 | 心身の健康上の問題から戦力とみなせない者 |
| ②勤務態度・協調性の欠如 | 心身に健康上の問題はないものの、勤務態度や協調性に問題があり戦力とみなせない者 |
| ③業務遂行能力不足 | 心身に健康上の問題はなく業務遂行意欲もあるものの、能力が足りず戦力とみなせない者 |
| ④能力と業務のアンマッチ | 上記に該当しないケースで、現職では能力発揮できないが、現職以外であれば職階相当の能力発揮が期待される者 |

**補足定義（画面に表示）：**
- 傷病＝療養休職中、あるいは要健康管理者で本人の為に特別に配慮された軽易な職務にしか従事できない者
- 傷病以外＝育児休業・介護休業・青年海外協力隊等のための依願休職
- その他＝上記に当てはまらず、戦力としてカウントできない者

### データ項目

| 項目 | 入力者 |
|------|--------|
| 対象者（メンバー選択） | 部長/課長 |
| 区分（①〜④） | 部長/課長 |
| 戦力外把握の理由（自由記述） | 部長/課長 |
| 戦力復帰時期 | 部長/課長 |

### 閲覧権限
- 入力者のライン上の上司 + 人事全員が閲覧可能
- 課長入力 → 部長・センター長・人事
- 部長入力 → センター長・人事

## 9. 画面構成の変更案

### 部長画面

| タブ | 変更内容 |
|------|---------|
| 転出・退職計画 | 変更なし（現行維持） |
| 転入・採用計画 | 充員理由・採用区分を追加。新卒詳細は2段階入力モーダル |
| 異動計画一覧 | ヘッダーに「バイネーム未確定 X名」アラート追加 |
| 承認状況 | 変更なし |
| **定員管理** | **新設**: 様式2相当の定員テーブル（課単位入力対応）+ 定員変動履歴 |
| **戦力外管理** | **新設**: 戦力外人員の登録・一覧（区分定義表示付き） |

### センター長画面

| タブ | 変更内容 |
|------|---------|
| 部員一覧 | 変更なし（新項目は追加済み） |
| 異動計画確認・修正 | 変更なし |
| **定員確認** | **新設**: 全部門の定員状況を横断確認 |
| **戦力外確認** | **新設**: 配下部門の戦力外人員を確認 |

### 人事画面

| 画面 | 変更内容 |
|------|---------|
| 人事最終承認 | 変更なし |
| **全社定員管理** | **新設**: 全組織の定員を俯瞰・CSV出力 |
| **戦力外一覧** | **新設**: 全社の戦力外人員を確認 |

## 10. 実装の優先順位案

| Phase | 内容 | 理由 |
|-------|------|------|
| **Phase 1** | 定員カテゴリ変更 + 組織4階層化 | 他の全機能の基盤になる |
| **Phase 2** | 充員計画の拡張（理由・採用区分・スペック追加） | 既存画面の拡張で対応可能 |
| **Phase 3** | 新卒採用詳細（2段階入力 + マスタ管理） | 新規画面・マスタが必要 |
| **Phase 4** | 戦力外管理（区分マスタ + 権限制御） | 独立した新機能 |
| **Phase 5** | 定員変動履歴 + バイネーム未確定アラート | 利便性向上 |
