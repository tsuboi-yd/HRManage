# 仕様変更提案 v2（確定事項まとめ）

## 確定した設計方針

### 1. 画面構成（タブ数は現行維持 + α）

**部長画面（6タブ → 5タブのまま）**
- 転出・退職計画 ← **戦力外列を追加**（別タブにはしない）
- 転入・採用計画 ← **充員理由・採用区分・新卒詳細を追加**
- 異動計画一覧 ← **バイネーム未確定アラートを追加**
- 承認状況（変更なし）
- 定員サマリ ← **上部固定のまま拡張**（棒グラフ削除、Excel様式2相当に）

**戦力外管理は独立タブにしない** → 部員テーブルの列として統合

---

### 2. 定員サマリの改修（上部固定カード）

#### レイアウト
棒グラフを削除し、テーブルを横幅フルに拡張。

```
┌──────────────────────────────────────────────────────────────┐
│ 定員・計画サマリ                               [一括申請]   │
├──────┬────┬──────────────┬──────────────┬──────────────┬──────┤
│      │    │  2026年度     │  2027年度     │  2028年度     │     │
│ 区分 │現員│ 定員│充員│過不足│ 定員│充員│過不足│ 定員│充員│過不足│
├──────┼────┼─────┼────┼──────┼─────┼────┼──────┼─────┼────┤
│ 正社員│ 14 │  30 │ +5 │  -1  │  35 │ +3 │   0  │  31 │  0 │
│ 管理職│  1 │   8 │  0 │  -1  │   7 │  0 │   0  │   7 │  0 │
│ 合計  │ 19 │  44 │ +7 │  -2  │  48 │ +3 │   0  │  44 │  0 │
├──────┴────┴─────┴────┴──────┴─────┴────┴──────┴─────┴────┤
│ バイネーム: 確定 5名 / 未確定 2名（2026年度）               │
└──────────────────────────────────────────────────────────────┘
```

#### 各列の算出ロジック

| 列 | ソース | 計算方法 |
|----|--------|---------|
| **現員** | メンバーマスタ | 群類で集計（管理職/正社員） |
| **定員** | 部長入力（編集可能セル） | クリックで編集、hover で変動理由表示 |
| **充員** | 採用計画（HiringPlan） | 該当年度の新卒＋キャリア合計を自動集計 |
| **過不足** | 自動計算 | 定員 − 現員 − 充員予定 |
| **バイネーム確定** | 異動計画 | transferType が設定済みのメンバー数 |
| **バイネーム未確定** | 自動計算 | 過不足のうち名前が入っていない枠 |

#### 定員変動理由
- 定員セルを編集した際、理由入力欄が表示される
- 前回の理由がプリセットされ、そのまま使えるし編集もできる
- **毎回ゼロから入力ではない**（前回理由を引き継ぎ）
- 保存後、定員セルにマウスホバーすると **ツールチップで理由を表示**
- 変更履歴はDBに保存（いつ・誰が・何を・なぜ変えたかを記録）

### 3. 転出・退職計画の戦力外列

部員テーブルに以下を追加：

| 列 | 内容 |
|----|------|
| 戦力外 | フラグ（ON/OFF トグル or チェックボックス） |

ONにした場合、インラインまたはポップオーバーで入力：
- **区分**（プルダウン: ①〜④）
- **理由**（自由記述）
- **復帰時期**（月選択、任意）

#### 区分マスタ（画面に定義表示付き）
| 区分 | 定義 |
|------|------|
| ① 健康・メンタル不調によるパフォーマンス不良 | 心身の健康上の問題から戦力とみなせない者 |
| ② 勤務態度・協調性の欠如 | 勤務態度や協調性に問題があり戦力とみなせない者 |
| ③ 業務遂行能力不足 | 能力が足りず戦力とみなせない者 |
| ④ 能力と業務のアンマッチ | 現職では能力発揮できないが、異動先では期待される者 |

補足定義も区分選択時にツールチップで表示：
- 傷病＝療養休職中、要健康管理者で軽易な職務にしか従事できない者
- 傷病以外＝育児休業・介護休業・青年海外協力隊等
- その他＝上記に当てはまらず戦力としてカウントできない者

#### 閲覧権限
- 課長入力 → 部長・センター長・人事が閲覧可
- 部長入力 → センター長・人事が閲覧可
- 組織ツリーの親方向に権限が伝播する仕組み

### 4. 採用計画の拡張

#### 基本項目に追加
| 追加項目 | 入力方式 |
|---------|---------|
| 充員理由 | 自由記述 |
| 採用区分 | プルダウン（既存事業維持/事業拡大） |
| スペック要件 | 自由記述 |
| 配属先（課） | プルダウン（任意、課単位入力時） |

#### 新卒採用の2段階入力

**Step1（基本情報）**— 必須、最初に入力
- 人数 / 配属先 / 時期 / 技術職or事務職 / 充員理由 / 採用区分

**Step2（詳細スペック）**— 任意、後から補完可能
- 事業部門 / 領域 / 専攻 / 学歴（B/M/D）/ NGの場合理由
- 語学力 / 資格 / 勤務地 / 職場環境 / 特記事項

#### キャリア採用の追加項目
- 具体的な業務内容 / 担当品 / 連絡先（人事担当）/ 入社時期 / 備考

#### プルダウンマスタ（管理者編集可能）
事業部門 / 領域 / 勤務地 / 専攻 / 採用区分 / 学歴

---

### 5. バイネーム未確定アラート

**表示箇所（両方）：**
1. 定員サマリテーブルの下段（常時表示）
2. 異動計画一覧画面のヘッダー部分にアラートバナー

**表示例：**
```
⚠ 2026年度: 2名の異動者が未確定です（定員枠に対してバイネーム未入力）
```

後で不要な方を削除する前提で、まず両方実装。

---

### 6. 組織4階層化

```
本部 → センター → 部 → 課（任意）
```

- Organizationモデルの level に 'section' を追加
- 部長が課単位入力するかは任意（部単位でもOK）
- 課単位入力時、部の定員は課の合計から自動集計
